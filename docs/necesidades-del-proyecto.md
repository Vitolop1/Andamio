# Andamio - Lo Que Necesito De Vos

## 1. Definicion del negocio

Necesito que me confirmes estas decisiones:

- nombre final del producto: `Andamio`, `Andamio Aula`, `Andamio Gestion` u otro
- quienes van a usar el sistema al inicio
- si el acceso de alumnos entra en el MVP o en fase 2
- si el sistema va a usarse solo en el consultorio o tambien en colegios

## 2. Usuarios reales

Necesito ejemplos concretos de usuarios:

- 1 administradora
- 2 o 3 profesionales tipo
- 2 o 3 alumnos tipo

Para cada uno necesito saber:

- que puede ver
- que puede editar
- que necesita hacer todos los dias

## 3. Flujo de trabajo real

Necesito que me cuentes como trabajan hoy:

- como guardan informes
- como guardan materiales
- como identifican a cada alumno
- como organizan los horarios
- que es lo que mas se pierde o desordena
- que quieren encontrar rapido apenas entran

## 4. Datos que queremos guardar

Necesito que definamos la ficha del alumno.

Minimo:

- nombre y apellido
- fecha de nacimiento
- colegio
- curso
- profesional asignada
- observaciones generales

Opcionales a decidir:

- DNI
- telefono familiar
- email familiar
- diagnostico o motivo de consulta
- obra social
- archivos medicos o externos

## 5. Tipos de archivo

Necesito una lista de archivos reales que hoy usan.

Por ejemplo:

- informes PDF
- imagenes
- actividades escaneadas
- planillas
- cuadernillos
- videos
- audios

Tambien necesito saber:

- tamano aproximado de archivos
- cuantos archivos pueden subir por semana
- si hace falta previsualizacion dentro de la web

## 6. Agenda

Necesito que definamos el modulo de horarios:

- si quieren vista diaria, semanal o ambas
- si manejan sesiones individuales y grupales
- si necesitan recordatorios
- si necesitan marcar asistencia
- si necesitan reprogramar sesiones

## 7. Portal alumno

Necesito decidir si el portal alumno arranca ahora o despues.

Si arranca ahora, necesito saber:

- el alumno entra solo o entra la familia
- que puede ver
- que puede subir
- si queremos seguimiento de entregas
- si queremos mostrar devoluciones o correcciones

## 8. Identidad visual

Necesito:

- logo
- colores de marca
- tipografia si ya existe
- referencias de paginas que les gusten
- estilo visual: mas institucional, mas calido, mas infantil o mas profesional

## 9. Operacion y costos

Necesito definir:

- si queres que usemos Supabase y Vercel
- presupuesto mensual aproximado para la plataforma
- cantidad estimada de usuarios en el primer ano
- cantidad estimada de alumnos cargados

## 10. Prioridades

Necesito que me ordenes estas funcionalidades del 1 al 5:

- alumnos
- archivos
- evaluaciones
- agenda
- instituciones y cursos
- portal alumno
- reportes

## Entregable ideal de tu lado

Si me mandas esto, ya podemos empezar a construir de verdad:

1. Logo de Andamio.
2. Nombre final del producto.
3. Lista de roles iniciales.
4. Lista de campos que debe tener la ficha del alumno.
5. Ejemplos de 5 archivos reales que hoy usan.
6. Capturas o fotos de como organizan hoy la informacion.
7. Prioridad exacta del MVP.

## Mi recomendacion para decidir rapido

Si queres avanzar sin frenar el proyecto, estas son mis recomendaciones iniciales:

- nombre: `Andamio`
- MVP sin portal alumno
- roles iniciales: `admin` y `profesional`
- storage: Supabase Storage privado
- despliegue: Vercel
- primera meta: alumnos + archivos + evaluaciones + agenda basica
